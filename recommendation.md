# Production Roadmap: Scaling to High-Scale Environments

**1. Data Persistence & Infrastructure Migration**
Given the timeframe and native-only constraints, this project utilizes thread-safe in-memory structures backed by a monolithic `snapshot.json` file for state persistence. While sufficient for small-scale checkpointing, overwriting a growing JSON file creates severe I/O bottlenecks. In a production environment, this JSON-based persistence must be replaced by a robust database (e.g., PostgreSQL or Cassandra) utilizing Write-Ahead Logging (WAL) for incremental state saving. Furthermore, the in-memory URL queue should migrate to a persistent message broker like **Apache Kafka**, and the "visited" set to a **Redis Cluster** equipped with a Bloom Filter to manage billions of URLs without exhausting RAM.

**2. Live Indexing & Relevancy Upgrades**
Currently, "Live Indexing" (searching while crawling) is achieved safely using local `threading.Lock` mechanisms. To scale this, the local index dictionary should be offloaded to a distributed search engine like **Elasticsearch**, which natively handles sharding and high-concurrency read/write operations. Finally, the relevancy heuristic must evolve from simple Term Frequency (TF). Implementing **TF-IDF** or **BM25**, alongside a pre-processing pipeline for stop-words and a graph-based algorithm like **PageRank**, will prevent common terms and low-authority domains from inflating the search results.
